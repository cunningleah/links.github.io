People often describe lock-free programming as programming without mutexes, which are also referred to as locks. Thats true, but its only part of the story. The generally accepted definition, based on academic literature, is a bit more broad. At its essence, lock-free is a property used to describe some code, without saying too much about how that code was actually written.
